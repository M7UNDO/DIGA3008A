Blog Post 8:Coding with Consciousness: Decolonialism and Design Decisions

Published on: 12 May 2025, by Mfundo Dhlamini

Reflecting on how my code matters in the context of decolonialism, and identifying things in my own coding practice that I should be aware of

When thinking about how my coding practice fits into a decolonial framework, one key thing I've become aware of is whose design conventions I'm centring and who gets excluded by default. Since most web development is shaped by Western-centric design systems, assumptions about user behaviour and accessibility standards don't always reflect local or Global South contexts. As I build my website, I ask myself if it reflects my culture and my community or if it just replicates templates made for someone else's audience.

Development process (Initial JavaScript implementation)

  • Scroll back to top button I added a scroll back to top button for certain pages on my website that have more content and require users to scroll down significantly. I realised that users would benefit from a quick way to scroll back up to the top, especially on mobile, where scrolling back to the top on long pages requires a lot of effort and excessive scrolling. Although this is a small change, it is something that has a meaningful impact on accessibility and comfort.
  • Navigation menu on smaller screens: I previously had issues with the navigation menu on smaller screens, as CSS alone wasn't flexible enough to animate the menu sliding in and out exactly as I wanted. However, by writing two functions, triggered by clicks on the burger and close buttons, which seamlessly open and close the navigation by adjusting the width of the navigation menu between 0% and 100%. This not only allowed me to achieve the sliding effect I envisioned but also simplified my code and improved the overall interaction design by making the menu more intuitive and responsive for mobile users.
  • Button mouseover/mouseout effects I wanted to create a hover effect that would invert the background and text colour of my button when the user hovered over it and then revert it when the mouse stopped hovering. JavaScript event listeners made this easy to implement using mouseover and mouseout events. This approach gave me full control over the interaction and helped me achieve exactly the visual feedback I envisioned. Combined with CSS transitions, it also improved the smoothness and quality of the user experience, making interactions feel more deliberate and polished.
